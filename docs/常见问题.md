# 常见问题

## 安装与集成

### Q: 为什么需要单独安装 cesium？

wiselayer 将 cesium 声明为 `peerDependency`，不会把 Cesium 打包进库中。这样可以：

- 避免包体积膨胀（Cesium 约几十 MB）
- 防止出现两个 Cesium 实例导致渲染异常
- 让使用方自行控制 Cesium 版本

### Q: Vite 项目中 Cesium 资源加载失败？

安装并配置 `vite-plugin-cesium`：

```typescript
import cesium from 'vite-plugin-cesium';
export default defineConfig({ plugins: [cesium()] });
```

同时引入样式：

```typescript
import 'cesium/Build/Cesium/Widgets/widgets.css';
```

## WMS 相关

### Q: WMS 图层不显示？

常见原因：

1. **跨域（CORS）** — 服务端未返回 Access-Control-Allow-Origin
2. **坐标系不匹配** — 检查 `crs` / `srs` 是否与服务端一致
3. **layers 名称错误** — 通过 GetCapabilities 确认图层名
4. **被其他图层遮挡** — 检查 `zIndex` 和透明度

### Q: WMS 1.1.1 和 1.3.0 有什么区别？

| 版本 | 坐标参数 | 默认 |
|------|----------|------|
| 1.3.0 | crs | wiselayer 默认 |
| 1.1.x | srs | 需设置 version: '1.1.1' |

wiselayer 会根据 `version` 自动选择 crs 或 srs 参数。

### Q: 如何叠加透明 WMS 到底图上？

```typescript
createLayer({
  type: 'wms',
  url: '...',
  layers: 'overlay',
  format: 'image/png',  // 支持透明
  zIndex: 1,            // 置于底图之上
});
layer.setAlpha(0.7);
```

## WMTS 相关

### Q: tileMatrixSetID 填什么？

需与服务端 WMTS Capabilities 文档中的 TileMatrixSet 标识一致。常见值：

- `EPSG:4326`
- `EPSG:3857`
- `GoogleMapsCompatible`
- `default028mm`（ArcGIS 服务）

不确定时，访问服务的 Capabilities URL 查看 XML 中的 `TileMatrixSet` 节点。

### Q: WMTS URL 支持哪些格式？

1. **KVP 编码**：`https://example.com/wmts?service=WMTS&request=GetTile&...`
2. **RESTful 模板**：`https://example.com/{TileMatrixSet}/{TileMatrix}/{TileRow}/{TileCol}.png`

## 定位（centerAt）

### Q: 点击定位提示「Layer has no bounded extent to center on」？

图层没有明确的地理范围（Provider 返回全球范围 `Rectangle.MAX_VALUE`）。

**解决：** 在配置中显式传入 `rectangle`：

```typescript
import { Rectangle } from 'cesium';

createLayer({
  type: 'wmts',
  // ...
  rectangle: Rectangle.fromDegrees(-180, -85, 180, 85),
});
```

### Q: centerAt 必须先 addTo 吗？

是的。定位依赖 Viewer 的 camera，图层必须先添加到 Viewer。

## 层级（zIndex）

### Q: zIndex 数值越大越靠上还是越靠下？

**越大越靠上**（后渲染，覆盖下层）。与 CSS z-index 语义一致。

- `0` = 最底层
- `1` = 在 0 之上

### Q: 修改一个图层的 zIndex 会影响其他图层吗？

会。Cesium 的 imageryLayers 是一个有序列表，移动一个图层时其他图层的 index 可能随之变化。示例项目中修改后会同步刷新所有图层的层级显示。

## 发布

### Q: npm publish 失败？

1. 确认已 `npm login`
2. 确认包名 `wiselayer` 未被占用
3. 确认版本号已递增（不能重复发布同一版本）
4. 运行 `npm run build && npm test` 确保构建通过

### Q: prepublishOnly 是什么？

`npm publish` 前自动执行 `build` 和 `test`，防止发布未构建或测试未通过的代码。

## 调试建议

1. 打开浏览器开发者工具 Network 面板，检查 GetMap / GetTile 请求是否 200
2. 查看请求 URL 中的 layers、crs、tileMatrixSet 等参数是否正确
3. 在 Cesium 中临时设置 `viewer.imageryLayers.get(0).show = false` 排查遮挡
4. 参考 `examples/vite-cesium` 对比配置差异
